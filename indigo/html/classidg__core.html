Detailed Description

A central repository for (volatile) information.

Note: While this class does not exactly complete the controller part of the MVC pattern - because it is stateless and provides no functionality of its own - it could easily be extended for this purpose. Things like cookies, session management, authentication and the like should indeed be handled by subclassing it.

Constructs an idg_core object and possibly loads a site template.

Site templates contain exactly one php script (and what is included from there, of course) defining a namespace that is subsequently used throughout the idg_core object.
Other than that, no assumptions are made as to their purpose or structure.

Notes:

  • The namespace can also be set programmatically. This is useful for generating files, e.g. from the command line.
  • The site designs that come with indigo are implemented as templates. Therefore, a site that uses such a design can have no template of its own.
